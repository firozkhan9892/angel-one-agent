Prompt Engineering

Vibe coding ka asli skill ye hai — kaise bolna hai AI ko. Ye formulas yaad rakhlo

📐

Basic Formula

[Kya banana hai] + [Kaise dikhe] + [Features]
Example: "Ek portfolio website banao, dark theme mein, glassmorphism cards ke saath, aur smooth scroll animation honi chahiye"
🚀

Advanced Formula

[Role assign] → [Context] → [Task] → [Constraints] → [Output format]
Example: "Tu ek senior React developer hai. Mera client ko ek dashboard chahiye jisme user analytics dikhe. Tailwind use karo, chart.js se graphs banao, mobile responsive hona chahiye. Complete code files do."
# E-commerce Page:
"Ek e-commerce product page banao. Hero section mein large product image, uske neeche features grid, phir reviews section, aur CTA button. Tailwind CSS use karo, dark theme, aur hover animations honi chahiye har element pe."
# Dashboard:
"Admin dashboard banao React mein. Left sidebar mein navigation, top mein search bar aur profile. Main area mein 4 stat cards (revenue, users, orders, conversion rate), neeche line chart aur recent orders table. Tailwind CSS, responsive design."
# Landing Page:
"SaaS landing page banao. Sections: Hero with gradient background + CTA, Features grid (6 features with icons), How it works (3 steps), Testimonials (3 cards), Pricing (3 tiers), Footer. Modern design, sky-blue accent color, smooth scroll."
# Too Vague:
"Website banao"
← Kis type ki? Kya features? Kaunse color?
# Too Technical (Wrong approach):
"Use flexbox with gap-4 and then add CSS grid for the cards section with grid-template-columns: repeat(3, 1fr)"
← Ye traditional coding hai, vibe coding nahi!
# Contradictory:
"Simple minimal design banao but bahut saare animations bhi daal do"
← Contradicting hai, AI confuse hoga.
# Generic Debug:
"Ye error aa raha hai: [paste error]. Isko fix karo aur batao kyu aa raha tha."
# Visual Bug:
"Mobile pe ye section overlap ho raha hai. Screenshot dekh ke fix karo. Breakpoint 768px pe adjust karo."
# Performance:
"Page load time 4 second hai. Unnecessary re-renders dhundh ke optimize karo. React DevTools mein kya check karu?"

Security & Pitfalls

Vibe coding mein galtiyan karne se pehle ye cheezein yaad rakhna — warna problem ho sakti hai

Kabhi Mat Karna

  • ✗
    API Keys Direct Code Mein
    AI kabhi kabhi .env use karke bhi keys directly code mein daal deta hai. Hamesha check karo.
  • ✗
    Production Mein Direct Deploy
    AI ka code pehle test karo. Security vulnerabilities ho sakti hain.
  • ✗
    Blindly Trust Karna
    AI galat bhi ho sakta hai. Code review karna seekho — kam se kam samajhna seekho ki kya likha hai.
  • ✗
    Client Data AI Ko Dena
    Kabhi bhi real user data ya sensitive info AI prompt mein mat daalo.

Hamesha Karna

  • ✓
    .env Files Use Karo
    Saare secrets .env mein rakhna. .gitignore mein .env add karna mat bhoolna.
  • ✓
    Code Review Karo
    AI ka code dhyan se padho. Kam se kam structure samajh lo.
  • ✓
    Input Sanitization Check Karo
    SQL injection, XSS — ye vulnerabilities AI code mein aksar rehte hain. Explicitly bolo "sanitize all inputs".
  • ✓
    Basic Code Samajhna Seekho
    Vibe coding ka matlab ye nahi ki coding seekhne ki zarurat nahi. Basic samajh hona chahiye taaki debug kar sako.
🔒

Security-Focused Prompt Template

"[Project description] banao. Important: Saare API keys .env mein rakhna, .gitignore mein .env add karna. Input sanitization karo (XSS, SQL injection prevent). Error messages mein sensitive info mat dikhana. Rate limiting add karo. Helmet/CORS properly configure karo."
